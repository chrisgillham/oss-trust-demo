These attacks are happening now
Not theoretical. Active campaigns.

Every incident below used a real package, a real maintainer, and real automated tooling to propagate. This demo shows exactly how the framework stops each one.

2026 · npm · CRITICAL
IronWorm — asteroiddao
36 npm packages infected with a Rust ELF binary dropped via preinstall hook. eBPF kernel rootkit, Tor C2, harvests OpenAI/Anthropic API keys, Exodus wallet seeds, AWS credentials. Unique encrypted payload per infection defeats hash-based IOCs.
2026 · npm · CRITICAL
Miasma — Red Hat Insights
32 packages backdoored via compromised employee GitHub account. Orphan commits bypassed PR review. id-token:write CI/CD permission used to publish via OIDC trusted publishing. Packages carried valid Sigstore signatures.
2026 · npm · CRITICAL
Shai-Hulud — TanStack
170 packages compromised using the same OIDC trusted publishing playbook. Revoking tokens made things worse — the attack mechanism doesn't depend on stored secrets.
2024 · tarball · CVSS 10.0
XZ Utils Backdoor
Two-year social engineering campaign. Attacker gained maintainer trust, injected a backdoor into build scripts. Shipped in major Linux distributions. SSH authentication bypass. The slowest, most patient attack in the dataset.
Value proposition
🔍

Catches attacks that valid signatures miss

IronWorm and Miasma produce legitimately signed packages. The framework audits the chain of custody behind the signature — orphan commits, PR review, publisher repo identity.

🧠

Behavior-based, not hash-based

IronWorm generates a unique encrypted payload per infection to defeat hash IOCs. Gate 5 matches on what the payload does — Tor connections, eBPF syscalls, AI API key access — not what it looks like.

⚡

Zero-day patches move fast

The 72-hour age hold is the highest-ROI control. The expedited lane bypasses only the age gate — with machine-verified CVE confirmation, 2-of-3 MFA quorum, and a 6-hour token TTL.

📋

Auditable by design

Every gate decision and exception emits a structured SIEM event. Ticket linkage is mandatory. Monthly retrospectives are enforced by circuit breaker. Built to satisfy auditors who weren't in the room.

🏗

Structurally hard to defeat

Each gate queries sources independent of the compromised repository. Defeating the framework requires compromising NVD, OSV, OpenSSF Scorecard, deps.dev, the npm attestation registry, and gVisor simultaneously.

🔌

Drop-in CI/CD integration

The GitHub Actions workflow fires on any lock file change, comments gate results on PRs, and fails the build on block or quarantine. No per-repo configuration after initial setup.

Installation
Up in five minutes.

The framework is not yet on PyPI — install it from the GitHub source repo directly, then set up the demo.

PowerShell
# 1. Clone both repos
git clone https://github.com/chrisgillham/oss-trust-framework
git clone https://github.com/chrisgillham/oss-trust-demo
cd oss-trust-demo

# 2. Create and activate virtual environment
python -m venv .venv
.venv\Scripts\activate

# 3. Install framework from local source
pip install -e "..\oss-trust-framework"
pip install rich click pytest pytest-asyncio pyyaml

# 4. Verify
oss-trust --version
# oss-trust, version 0.2.0

name: OSS Trust Demo - CI

  trust-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.12"
      - name: Install framework from GitHub
        run: |
          pip install "git+https://github.com/chrisgillham/oss-trust-framework.git"
          pip install rich click pytest pytest-asyncio pyyaml
      - name: Validate requests 2.33.0
        run: |
          oss-trust check \
            --package requests --version 2.33.0 \
            --ecosystem PyPI --github-repo psf/requests \
            --output json | tee trust.json
      - name: Run 131 tests
        run: pytest tests/ -v
⚠
If you see ModuleNotFoundError: No module named 'oss_trust_framework'

Make sure oss_trust_framework/cli.py exists in the framework repo containing from oss_trust_framework.pipeline.cli import main, and that pyproject.toml has packages = ["oss_trust_framework"]. Then reinstall with pip install -e . from the framework directory.

Option 1 — Add to allowlist

Adds the package to config/trusted_publishers.yaml. Gate 2 will now verify that provenance attestations point to the canonical GitHub repo. A mismatch is a BLOCK.

Option 2 — Allowlist + require_attestation

Strongest protection. A missing attestation is upgraded from QUARANTINE to BLOCK. Use for your most security-critical dependencies: crypto primitives, HTTP clients, auth libraries.
